# EAC Tracker — Improvements Log

This document summarises improvements applied during the 12 Jun 2026 audit of the
EAC Tracker app against the SAP `ZPSR0021A` report (`Sample.xlsx`). For each
change it records **what** was done and **why** it was needed.

## 1. SAP importer — dedupe continuation pages

**What.** Rewrote `parseWorkbook` in [backend/src/routes/sap.js](backend/src/routes/sap.js) to key projects by `sap_project_no` in a `Map`. Repeated `"Project"` header rows for the same WBS now resolve to the same record; sub-jobs are merged with a `Set`-based dedupe on `wbs_code`; header-level fields are back-filled from later occurrences only when the first occurrence was missing them.

**Why.** The sample workbook contains the project header for `214876532/FS/001` (NG PROJECT) **9 times** and `203451789/004` twice — SAP repeats the header on every continuation page. The previous parser pushed each block as a separate project, producing `project_count: 16` from a 7-project file and causing the same row to be UPSERTed multiple times (the last write winning, with all but one being wasted work and the audit log polluted).

## 2. SAP importer — Total row is authoritative

**What.** The `"Total"` row at the end of a project block now always overwrites `current.totals`. The earlier "project-summary" row (WBS without suffix) is kept as a fallback only when no `Total` row has been seen.

**Why.** Both rows exist in every block, but the project-summary row appears *before* the Total row. After change #1 the dedupe logic was treating the summary row as the first totals it saw and refusing to overwrite at the Total row — producing wrong actuals (e.g. `214687801/035` showed actual = `$1.85M` instead of the correct `$9.46M`). SAP's `Total` line is the authoritative aggregate.

## 3. SAP importer — auto-derive portfolio status

**What.** Added `deriveStatus(budget, eac)` to map EAC variance to the `status` enum (`'ok' | 'warn' | 'bad'`):

| `|variance| %` | status   | label    |
| -------------- | -------- | -------- |
| `≤ 10%`        | `ok`     | On Track |
| `> 10%, ≤ 25%` | `warn`   | At Risk  |
| `> 25%`        | `bad`    | Delayed  |

Both the INSERT and UPDATE branches of `/api/sap/commit` now write the derived status.

**Why.** Every imported project was defaulting to `'ok'`, so the Portfolio screen showed "7 on track" even when projects had +99.7% or −45.5% variance. PD/Finance had no visual signal of which projects were off-rails.

## 4. SAP importer — keep EAC in sync with SAP totals

**What.** The UPDATE branch now also sets `eac = (actual + committed)::numeric`, matching the INSERT branch.

**Why.** The UPDATE previously omitted `eac`, so successive monthly imports left `eac` showing the value from the original CREATE while `actual` and `committed` moved on. This caused the derived status (computed from `tot_cost + com_cst`) to disagree with the variance shown to users (computed by `v_project_financials` from the stale `eac` column). After the fix, the displayed variance, the derived status, and the SAP totals all match.

## 5. SAP importer — Responsible Person → PM auto-assignment

**What.** Parser now captures `responsible_id` (col 21, SAP employee number) and `responsible` (col 22, name). Added `users.sap_employee_id BIGINT UNIQUE` column. On commit a `resolvePm()` helper looks up the user by `sap_employee_id`, then by case-insensitive `full_name` (back-filling the SAP id on a match), and otherwise auto-creates a `PM` user with a derived `f.lastname` username. The resolved `pm_user_id` is written to the project on both INSERT and UPDATE (UPDATE uses `COALESCE` so a NULL doesn't wipe an existing assignment).

**Why.** Before this, every SAP-imported project had `pm_user_id = NULL` and showed "(Unassigned)" in the Portfolio, so the PM-Dashboard role filter returned 0 projects for every PM. The sample workbook already carries the PM name in the header row, so manual assignment was unnecessary friction.

## 6. SAP importer — end-date column fallback

**What.** `parseDdMmYyyy(row[37] || row[36])` instead of just `row[36]`.

**Why.** In the sample file the End Date value lives at column 37; column 36 is `null`. The previous parser silently produced `end_date = null` for every project.

## 7. Portfolio header reflects signed-in user

**What.** [frontend/src/screens/Portfolio.jsx](frontend/src/screens/Portfolio.jsx) subtitle now reads `"{My projects | All projects} · {session.full_name} ({role title})"` instead of the hard-coded `"All projects · K. Rajah (Project Director)"`.

**Why.** The hard-coded subtitle made it look like the page was scoped to K. Rajah regardless of who was actually signed in — confusing and demo-y.

## 8. Login screen — sorted user list & auto-refresh

**What.** [frontend/src/screens/Login.jsx](frontend/src/screens/Login.jsx) now sorts the quick-pick list by role priority (Admin → Finance → PD → PM) then alphabetically, and re-fetches `/api/users` on window focus.

**Why.** With auto-created PMs the list grew unsorted. Sorting groups equivalent roles together. The focus-refresh means newly-created PMs (e.g. after a SAP import in another tab) appear on the Login screen without a manual page reload.
